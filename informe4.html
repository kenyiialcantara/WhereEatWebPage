Requerimientos no funcionales

1. Carga rápida

La aplicación debe cargar rápidamente para ofrecer una experiencia fluida a los usuarios y minimizar los tiempos de espera.
Para lograr alcanzar este requerimiento, se pueden realizar las siguientes configuraciones en la interfaz:

  • Comprimir y ajustar el tamaño de las imágenes y archivos multimedia para reducir el tiempo de carga.
  • Implementar técnicas de almacenamiento en caché para almacenar en el dispositivo del usuario los elementos estáticos de la aplicación, lo que permite cargarlos más rápido en visitas posteriores.
  • Minimizar y comprimir los archivos de código (HTML, CSS, JavaScript) para reducir su tamaño y mejorar la velocidad de carga.
  • Cargar inicialmente solo el contenido visible en la pantalla y luego cargar de manera progresiva el contenido adicional

2. Seguridad

La aplicación debe garantizar la seguridad de la información del usuario, como datos personales y de pago, mediante la implementación de encriptación de datos.
Al estar desarrollando la aplicación usando Firebase, se provee de mecanismos de seguridad inherentes al acceso de cuentas a través de Google, como los siguientes:

  • Autenticación de cuentas seguro
  • Almacenamiento de datos en la nube, protegido por Google
  • Protección contra ataques, mediante filtros y validaciones implementados por Firebase
  • Gestión de identidades y roles, lo que evita que usuarios no autorizados realicen acciones fuera de sus capacidades.

3. Accesibilidad

La aplicación debe ser accesible para usuarios con discapacidades, brindando una experiencia inclusiva y facilitando su uso.
Se planea usar interfaz de control de voz para poder navegar a través de la aplicación, por lo que se deben cumplir los siguientes criterios:

  • Etiquetar debidamente a través de HTML los elementos interactivos de la página, para que sean fácilmente descritos al usuario y seleccionados mediante comandos por voz.
  • La estructura de navegación de la aplicación debe ser clara y lógica, lo que permitirá a las personas con discapacidad visual parcial moverse de manera eficiente entre las diferentes secciones y funciones de la aplicación utilizando comandos de voz.
  • Proporcionar opciones de personalización para adaptar la interfaz a las preferencias individuales de los usuarios. Esto puede incluir ajustes de velocidad de la voz, preferencias de lectura y opciones de navegación específicas para satisfacer las necesidades de las personas con discapacidad visual parcial.

4. Escalabilidad

La aplicación debe ser capaz de manejar un aumento en el número de usuarios y transacciones sin degradar el rendimiento.

  • Firebase proporciona una infraestructura escalable en la nube, lo que significa que la aplicación puede adaptarse automáticamente a los picos de demanda y al aumento del tráfico sin intervención manual.
  • Además, incluye el Firebase Cloud Storage, un servicio de almacenamiento en la nube que permite almacenar y servir contenido estático, como imágenes, videos y archivos. Este almacenamiento se escala automáticamente para adaptarse a la demanda, asegurando una entrega rápida y confiable del contenido a los usuarios.
  • Adicionalmente, también cuenta con centros de datos alrededor del mundo, lo que permite que la aplicación se pueda escalar y esté disponible en muchas regiones del mundo de manera rápida.

5. Usabilidad

La aplicación debe ser fácil de usar y proporcionar una experiencia intuitiva y agradable para los usuarios.
Para lograr este requerimiento, se debe:

  • Realizar pruebas de usabilidad y recopilar comentarios de los usuarios para iterar y mejorar continuamente la interfaz de usuario.
  • Establecer una estructura de navegación lógica y coherente que permita a los usuarios moverse fácilmente por la aplicación y encontrar la información o las funciones que necesitan.
  • Adaptar la interfaz de usuario a diferentes dispositivos y tamaños de pantalla para garantizar una experiencia consistente
  • Permitir a los usuarios personalizar ciertos aspectos de la aplicación

Requerimientos Funcionales

1. Registro de usuario (Público) en la aplicación

Requerimientos:

El sistema debe permitir a los usuarios crear una cuenta en la aplicación.
Los usuarios deben proporcionar información como nombre, dirección de correo electrónico y contraseña para registrarse.
El sistema debe validar la información proporcionada por el usuario durante el proceso de registro.
El sistema debe almacenar de forma segura la información del usuario en la base de datos.

Caso de uso:

  1. El usuario abre la aplicación e inicia el proceso de registro.
  2. El sistema muestra un formulario de registro que solicita información como nombre, dirección de correo electrónico y contraseña.
  3. El usuario completa el formulario con la información requerida.
  4. El sistema valida la información ingresada por el usuario.
  5. Si la validación es exitosa, el sistema almacena de forma segura la información del usuario en la base de datos y genera una cuenta para el usuario.
  6. El sistema muestra un mensaje de confirmación al usuario, indicando que el registro se ha completado con éxito.
  7. El usuario puede iniciar sesión en la aplicación utilizando las credenciales proporcionadas durante el registro

El sistema debe permitir a los usuarios buscar restaurantes cercanos a su ubicación actual.
El sistema debe utilizar la ubicación del usuario para mostrar los restaurantes más relevantes en el área.

Caso de uso:

14. Navegación por voz

Caso de uso

Interfaz de voz para navegación en la página web, mediante comandos de voz procesados y enviados como peticiones a la API de ChatGPT, el cual responde un objeto con las posibles respuestas al comando, que se leen mediante un lector de pantalla

Descripción

Implementar una funcionalidad que permita a los usuarios navegar por la página web utilizando comandos de voz en lugar de interactuar con elementos visuales. Los usuarios podrán realizar acciones como seleccionar opciones del menú, acceder a diferentes secciones de la página, realizar búsquedas, interactuar con elementos interactivos y completar tareas básicas utilizando únicamente su voz.

La navegación por voz ofrece una experiencia accesible e inclusiva para personas con discapacidad visual o dificultades de motricidad, permitiéndoles interactuar y utilizar la página web de manera eficiente y cómoda. Además, proporciona una alternativa conveniente para todos los usuarios, ofreciendo una experiencia de navegación hands-free que puede ser más rápida y práctica en ciertos contextos, como al realizar tareas mientras se encuentran ocupados con otras actividades.

Criterio de cumplimiento

  • Los usuarios pueden navegar por la página web utilizando comandos de voz de manera intuitiva y precisa.
  • Los comandos de voz son reconocidos de manera confiable y responden con acciones apropiadas.
  • La navegación por voz es equivalente en funcionalidad y eficiencia a la navegación visual con elementos interactivos.

Dependencias

  • Implementación de un sistema de reconocimiento de voz robusto y preciso.
  • Diseño y desarrollo de comandos de voz específicos y comprensibles para las acciones y elementos clave de la página web.

Conflictos

  • Posibles conflictos con otras funcionalidades o interacciones de la página web que podrían entrar en conflicto con los comandos de voz, como eventos de clic o acciones programadas.
    Fuente: Los requerimientos se derivan de la necesidad de ofrecer una experiencia accesible y práctica para los usuarios de la página web.

  • Tiempo estimado de cumplimiento: El tiempo de implementación puede variar según la complejidad de la página web y las funcionalidades existentes. Se requerirá un análisis y diseño detallado antes de la implementación, por lo que el tiempo estimado puede variar entre una y tres semanas, dependiendo de los recursos y el alcance del proyecto.

15. Chatbot LLM (Language Model)

Caso de uso

Implementación de un chatbot basado en el modelo de lenguaje GPT (Generative Pre-trained Transformer) para brindar respuestas a preguntas y llevar a cabo conversaciones con los usuarios de la página web.

Descripción

El chatbot LLM utiliza el modelo de lenguaje GPT para generar respuestas contextuales y coherentes a partir de las consultas y mensajes de los usuarios. El objetivo principal es proporcionar una experiencia interactiva y conversacional en la página web, donde los usuarios puedan obtener respuestas a sus preguntas, recibir recomendaciones, realizar consultas y obtener información de manera eficiente y natural.

El chatbot se integra en la página web y se muestra en una ventana de chat. Los usuarios pueden interactuar con el chatbot ingresando mensajes de texto y recibiendo respuestas en tiempo real. El chatbot utiliza técnicas de procesamiento de lenguaje natural para comprender las consultas de los usuarios y generar respuestas relevantes.

Criterio de cumplimiento

  • El chatbot LLM brinda respuestas relevantes y coherentes a las consultas de los usuarios.
  • El chatbot comprende y maneja una variedad de preguntas y consultas relacionadas con el dominio - específico de la página web.
  • El chatbot proporciona una experiencia interactiva y conversacional, respondiendo de manera natural y fluida a los usuarios.
  • El chatbot es capaz de aprender y mejorar su rendimiento con el tiempo.

Dependencias

  • Implementación del modelo de lenguaje GPT o un modelo similar basado en transformers.
  • Integración del chatbot en la página web, incluyendo la interfaz de usuario para ingresar consultas y recibir respuestas.

Conflictos

  • Posibles conflictos con otras funcionalidades o interacciones de la página web que podrían entrar en conflicto con el chatbot, como eventos de clic o acciones programadas.
  • La precisión y relevancia de las respuestas del chatbot pueden variar según la calidad y cantidad de datos de entrenamiento disponibles.
  • El chatbot puede enfrentar dificultades para comprender preguntas ambiguas o poco claras.
